1.0 · The Engine

A hash engine, written from packet captures.

The chain wasn't going to write its own miner. So we did — in C, hardware-aware, one binary, three platforms, sixty-second install. Custom Scrypt with the post-ROMix transformation the network actually validates. Not a Litecoin fork with a logo swap.

1.1

Custom Scrypt + post-ROMix

N=1024 r=1 p=1 with the BlockDAG transformation. Pure C. Native compile per chip — x86, ARM64, Apple Silicon.

1.2

Hardware-aware installer

Detects CPU, cache topology, NVIDIA GPU. Auto-tunes thread count, pins to P-cores, configures huge pages.

1.3

Built-in dashboard

Embedded HTTP server on :8881. Hashrate, accepted shares, per-thread breakdown. No third-party agent required.

1.4

Stratum v1 + vardiff

Complete protocol implementation. Auto-reconnect, clean job switching, nonce range management. Pool-agnostic.

2.0 · The Mesh

A pool that never blinks.

Three pools, three schemes, one production backend. WireGuard tunnels into AWS af-south-1 and us-west-2 take the miner-side latency hit, then bridge into the UAE pool over an encrypted line. Sub-300ms submission latency regardless of which continent you sit on.

2.1

Three pools, three schemes

asic-pool · PROP · 3334. cpu-gpu-pool · PPLNS · 3335. hans-solo · SOLO · 3336. Independent accounting, shared infrastructure.

2.2

Sync-aware failover

HAProxy with two-stage health check — getBlockCount alive plus getBlockTemplate not in initial-download mode. Lagging nodes drop themselves.

2.3

WireGuard relays

AWS af-south-1 (Cape Town) and us-west-2 (Oregon). DNAT 3334/3335/3336 into the encrypted tunnel. Live packet rate, live handshake age, observable to the millisecond.

2.4

Live mesh map

Every connected miner plotted to IP geolocation, great-circle routed to their entry relay. D3 + Natural Earth, refreshed every minute. Privacy-preserving.

4.0 · The Index

A block explorer that actually indexes.

Full-fat Blockscout backend, our own indexer pipeline against our own node fleet, public RPC fronted by HAProxy with sync-aware health checks. The chain's data layer, served at near-zero latency from Cape Town to Oregon.

4.1

Blockscout indexer · midnight theme

Blocks, transactions, addresses, tokens, internal txns, contract verification, decoded calldata. Standard tooling for our chain.

4.2

Public JSON-RPC

rpc.dagtech.network — HAProxy 4-backend failover, nginx LRU cache on eth_getBalance, two-stage sync health gate. Free for the community.

4.3

Netstats feed

Live chain head visualisation. Block production cadence, peer count, fork detection. The whole chain at a glance.

5.0 · The Watch

Every system, observable.

Eight versions of the ops dashboard later, we settled on this: a topology-driven JARVIS panel reading from one canonical status.json that the entire stack reports into. Grafana for time-series, Telegram for state-change alerts, the mesh for the public surface.

5.1

One source of truth

excalibur-topology.py runs every 60s, emits /var/www/miners/v6/data/status.json. Every surface reads from this. No drift.

5.2

JARVIS panel

Live SVG topology overlay, dependency-aware edge routing, X100 hashrate trace from the 250-sample cpb_hshistory buffer.

5.3

State-change alerting

Telegram fires when components_bad crosses zero — not every 60 seconds. Cross-watcher correlation = SEV-1.
