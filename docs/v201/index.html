4New flags

--rescue

Non-destructive recovery mode. Replaces the destructive --cleanup advice that wiped chain data on operators following misleading runbooks. Soft-clears stale shutdown breadcrumbs without touching the trie, then attempts normal start.

--state.diff-layers-max=N

Caps in-memory PathDB diff layers. Default 256. Lower (128) for memory-constrained hosts, raise (512+) for aggressive ingest hosts with abundant RAM. v1 hardcoded this to 128 and never surfaced it.

5What v2.0.1 fixes

  • "Illegal withdrawal at block X" is now recognised as a shutdown breadcrumb (not a withdrawal). --rescue soft-recovers without wiping the chain.
  • "missing trie node" after unclean shutdown is auto-recovered on next start instead of refusing to boot.
  • maxDiffLayers is no longer hardcoded. Adapts to DAG reorg depth via --state.diff-layers-max.
  • PathDB flush / snapshot / reorg transient errors are now distinguished from corruption. The node retries rather than panicking.
  • lockfile orphan on container kill no longer prevents restart; replaced by atomic PID-tracked lock.
  • Reproducible builds: SHA-256 signed, deterministic across rebuild. 10/10 reproducibility verified.

6Chain snapshot (skip the resync)

A full resync from genesis takes ~2.7 days. Instead, the installer pulls Jeremy's auto-updating IPFS chain snapshot - SHA-verified, peer-list-bundled, ~10 GB compressed. Updated continuously.

# Pull latest snapshot manually (the installer does this automatically)
ipfs get /ipns/blockdag-snapshot.dagtech.network --output=./snapshot.tar.zst

# Or via HTTP gateway if IPFS isn't installed
curl -fLO https://ipfs.io/ipns/blockdag-snapshot.dagtech.network/latest.tar.zst
i

Use the subdomain gateway form (CID.ipfs.dweb.link/) if a path-style gateway returns 404.

7Upgrade from v1

One command. Stops v1 cleanly, snapshots the datadir, installs v2.0.1, restarts, verifies first block import within 60 s; rolls back on failure.

curl -fsSL https://miner.dagtech.network/v201/upgrade.sh | bash
i

Your v1 datadir is renamed to <datadir>.v1-backup-<timestamp> - not copied. The upgrade is atomic and reversible.
