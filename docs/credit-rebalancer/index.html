CRITICAL POOL FIX

Credit Rebalancer

If you operate a BlockDAG mining pool using pool-stack-docker-pool-v1.3.23 or any earlier asic-pool binary, your high-difficulty ASIC miners are being massively under-credited and low-difficulty CPU/GPU miners are being massively over-credited. This open-source sidecar fixes that — in-database, reversible, with a complete audit trail.

Install Guide View the Code

The Bug

The compiled mining-pool Go binary computes credits as inversely proportional to share difficulty:

credit_i  proportional to  1 / share_difficulty_i      // wrong
credit_i  proportional to  share_difficulty_i          // correct PPLNS

Observed live on a mixed pool with three connected workers (2026-06-18):

MinerVardiffWrong (Before)Correct (After)
Low-diff CPU/GPU0.5 to 1093.4%0.4%
Medium-diff ASIC~506.0%8.1%
High-diff ASIC (X30 etc.)thousands0.5%91.4%

The ASIC owner contributing about 99% of the actual hash work was receiving about 0.5% of the rewards. Silent value transfer between miners; operators are often unaware.

The Fix

A small Python sidecar container scans the credits table every 30 seconds for is_paid=false rows and applies the inverse-of-inverse correction per block:

correct_amount_i = R * (1 / wrong_amount_i) / SUM(1 / wrong_amount_j)
        where R = SUM(wrong_amount_j)  // the original block reward

This cancels the bug exactly while preserving the total block reward bit-for-bit. Every change is logged to a credit_rebalance_audit table.

Safety

  • Reversible: systemctl disable --now dagtech-credit-rebalancer.service
  • Audit table created on first run; consider snapshotting credits before enabling
  • Invariant preserved: total block reward stays exact
  • No binary patching; works alongside the existing asic-pool

Install in 60 seconds

git clone https://github.com/Dagtechltd/dagtech-pool-stack
cd dagtech-pool-stack/dagtech/credit-rebalancer
sudo ./install.sh

The installer auto-detects your pool's docker network and database URL, builds the Docker image, writes /etc/dagtech-credit-rebalancer.env, and enables a systemd unit.

Verify

systemctl status dagtech-credit-rebalancer.service
journalctl -u dagtech-credit-rebalancer.service -f

Within 30 seconds you should see lines like REBALANCED block=... changes=[...] if you have multiple miners. With a single miner per block, the rebalancer correctly does nothing (the bug has no observable effect there).

Notes for Pool Operators

  • If you have already paid out credits with the wrong math, that is done — the rebalancer can only correct credits with is_paid=false. Consider issuing a goodwill top-up if your ASIC customers were affected long-term.
  • This is a workaround. A patched mining-pool binary needs to flip the weighting at asicPool/internal/pool/pplns.(*Window).PushShare and .Snapshot. We are contributing the diagnosis upstream.
  • The rebalancer is safe to leave running indefinitely. When a corrected binary lands, the rebalancer becomes a no-op.
